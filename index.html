Object persistence

CoreObject is a version-controlled object database for Objective-C, designed to be a humane persistence layer for applications with a "never lose any work" philosophy.

User Experience

CoreObject came from imagining how we would apply first principles of UI design to graphics and productivity applications in a legacy-free world.

Simple

  • no save command, and never lose even one keystroke
  • we can go back anywhere in the past instantly
  • document variations or branches (and possibility to convert copies into branches and vice-versa)
  • the history can be navigated at various granularities (document, activity etc.)

Resilient

  • everything can be undone even deletion, copy, undo etc.
  • any change can be cancelled individually in the history (no matter how big is the history)
  • undo/redo persistent across application restart
  • merging support that doesn't require use to look at a textual merge
Our idea is to go futher than traditional filesystems by including document/data structure and organization, collaborative editing, rich undo/redo, and version control directly in CoreObject, to allow all applications to easily implement these features, and avoid each app reinventing the wheel in incompatible ways. In existing apps, we either see poor persistence support, or reliance on proprietary systems.

Open

  • collaborative editing
  • concurrent editing accross multiple applications, with updates propagated transparently to every working copies in memory
  • easy data sharing accross multiple applications
  • everything is searchable in the present and in the past
  • a UI toolkit or compound document model can be written on top

Features

CoreObject is a pragmatic OODB built on top of a SQL database that rethinks ideas from various horizons.

Version Control

  • Distributed version control based on an object graph diff model
  • Structured data merge works well for most use cases; pluggable merge strategies (with an example styled text merge algorithm)
  • Pervasive undo/redo
  • Muliple undo or history views over one or several documents
  • Selective undo (each change can be cancelled individually)

Object Model

  • Versatile object model with rich relationship support
  • Flexible storage model with documents and branches backed by a store (documents are similar to Git repositories)
  • Stable serialization format with integrity checking
  • Optional organizational model with tags and libraries

And More…

  • Relationships accross documents and branches
  • Transparent object graph copy
  • Document cheap copy (a copy consumes no space)
  • Real-time collaborative editing with full undo support per user
  • Built-in indexing and search including over the history

Framework

CoreObject has slowly evolved to its current shape. Almost a decade was spent on writing it, we started discussing the ideas behind it back when Étoilé was founded.

Lean

  • Written in Objective-C with ARC
  • Minimal dependencies (SQLite and EtoileFoundation)

Polished

  • Small code base (around 20 000 loc)
  • Reasonably large test suite (around 12 000 loc)
  • Good debugging support (e.g. render object graph as PDF diagram from debugger)
More than seven prototypes have been written to explore various design choices, until we settled on the current architecture. Although it is still a preview release, the core architecture and API are pretty mature and shouldn't undergo any major changes.
